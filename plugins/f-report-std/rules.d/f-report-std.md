# 报告写作规范（全局约束）

> 完整模板+工作流 → `f-report-std` skill。4 套模板 → `templates/`。
> 飞书格式硬约束（lark-table 822/无手动编号）→ `f-doc` skill。

## 结构
- H1 顶级，每章 2-3 句概括说明
- 不加手动编号（飞书自动生成目录）
- 章节顺序：执行摘要 → 分析框架 → 逐章展开 → 对比/矩阵 → 关键结论
- "关键结论" / "总结"章节也用 H1

## 数据呈现
- 对比场景（X vs Y）→ 表格/矩阵，禁纯列表
- 多层级架构图 → 每层横向并列
- 数值用实际量级（万元级非千元级），与真实用量匹配

## 论证
- 每章：数据支撑 + 因果链条 + 不确定项标注
- 通用性表达：使用"该企业""目标场景"等泛指，不写具体企业名

## 引用
- 国标/标准引用完整不简化（如三级体系只用到两级也要说全三级）

## 图表（图子文档约定）
- 父文档作图 → 建独立子文档（命名 = 图名）
- 子文档结构：图 + 解读（含分析） + 代码
- 父文档用 `block_insert_after` 嵌入图块到指定位置
- 子文档 = 详细档案，父文档 = 摘要嵌入
- 详见 `f-doc` skill 工作流 G
